# ReSale Agent 路线图

## 当前判断

V1 已经跑通主链路：图片识别、用户确认、动态追问、本地相似商品、规则估价、出售文案、Markdown 导出和模拟议价。V2 已围绕“更可解释、更可复用的出售辅助”完成收口。V3 采用本地闭环方式推进个人闲置库存、发布后表现反馈和授权价格样本管理，不接真实平台、不爬取数据。

## V1 收口清单

- 保持三类商品范围：数码产品、图书、小家电。
- 保持本地 SQLite 模拟商品库，不接入真实二手平台。
- 保持估价由规则计算，大模型只做识别和文本生成辅助。
- 补齐新环境启动、CI、固定演示案例和 README 说明。
- 所有新增能力必须能在没有模型密钥时安全降级。

## V2 已完成增量

- 增强估价解释：展示规则估价、市场中位价、样本数量和融合权重。
- 增加轻量草稿管理：保存历史会话、重新打开草稿、删除草稿。
- 增加多平台风格文案：生成闲鱼、转转、小红书风格文案，但不触发真实发布。
- 增加成交反馈记录：保存最终成交价、成交渠道和估价偏差。
- 增加成交反馈汇总视图：展示成交数、分类命中比例和最近成交偏差，并支持按类别/渠道筛选。
- 增加服装类别：扩展服装模拟数据、字段追问、文案和演示案例。
- 增加家具类别：补充家具尺寸、拆装、搬运信息追问和本地成交样本。
- 增加鞋包类别：补充鞋码、磨损清洁、购买渠道/验货信息和本地成交样本。
- 增加本地图片相似度检索原型：用上传图片签名、原始文件名和识别线索辅助排序模拟相似商品。
- 增加调价明细展示：拆出折旧、成色、状态调整、市场融合和发布建议的阶段影响。
- 增加发布前检查清单：提醒补齐价格、瑕疵、照片、交易方式和品类关键说明。

## V2 收口状态

- V2 已完成：解释型估价、多品类支持、草稿管理、多平台文案、成交反馈、复盘筛选、本地图片相似检索、调价明细和发布前检查。

## V3 已完成增量

- 增加个人闲置库存：基于本地会话维护草稿、待发布、已发布、已成交和已归档状态。
- 增加库存总览：支持按类别和库存状态查看个人闲置物品。
- 增加发布后表现反馈：记录发布天数、浏览量、收藏量、咨询量和当前挂牌价。
- 增加本地调价建议：根据表现信号给出继续观察、保持价格、优化展示或小幅降价建议，并守住最低接受价。
- 增加授权价格样本导入：通过 CSV 将用户自有或授权成交样本加入本地相似商品库，并在估价解释中标明导入样本数量。
- 增加价格样本管理：查看样本来源汇总、按类别/来源/启用状态筛选，并允许停用、恢复、备注或删除误导入的授权样本。
- 增加价格样本审计：记录导入、停用、恢复、备注和删除操作，删除样本后仍保留本地审计记录。
- 增加审计导出：支持将价格样本操作审计导出为 CSV 或 Markdown 备份。

## 后续候选方向

- 增加价格样本批量管理。
- 在明确授权和合规边界内适配真实平台发布流程，但不进入支付、物流和售后。

更细的后续开发拆分见 [next-development-plan.md](next-development-plan.md)。

## 不做事项

- V1 不做真实平台登录、自动发布、真实买家聊天、支付、订单、物流和售后。
- V1 不做微服务、消息队列、Kubernetes 或复杂权限系统。
- V1 不把价格交给大模型直接决定。
